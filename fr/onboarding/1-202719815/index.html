Instructions pour l’installation 

Étape 1 : Télécharger le fichier .exe depuis notre site internet.

Étape 2 : Guide de la liste blanche de Reacts 

Doit être autorisé sur les ports 443 et 5349, pour les protocoles TCP et UDP.

Liste blanche pour le filtrage au niveau HTTP

Si vous utilisez un filtrage de niveau HTTP dans votre pare-feu / proxy, assurez-vous que le protocole WebSocket est autorisé.

Règle optionnelle
La règle suivante n’est pas nécessaire au fonctionnement de la plate-forme Reacts, mais elle est recommandée car elle améliorera les performances de streaming.
Pour améliorer performances de streaming, assurez-vous que les connexions entrantes sur la plage de ports 49152-65535 sont autorisées pour le protocole UDP.

Le diagnostic des problèmes de connectivité

Voici les étapes de diagnostic pour vous aider à résoudre quels problèmes standard de connectivités

Étape 1 : Vérifier que votre pare-feu et ou vos filtres webs autorisent Reacts à se connecter,  accédez à l’adresse https://svc.iitreacts.com/api/echo au moyen de votre navigateur.

 Le message «{“Version”:””,”Body”:”Echo OK!”,”Type”:”System.String”,”Time”:”2016-07-27T15:04:59.6325394Z”,”Id”:”676b0cf1-e3ad-4cb7-bb9d-6076e2913efc”}» devrait s’afficher.

Étape 2 : S’assurer qu’il n’y a pas d’interférences entre votre logiciel Antivirus et Reacts qui pourraient empêcher le lancement du logiciel.

Étape 3 : Entrer les informations concernant votre Proxy. Si vous vous connectez via un Proxyet que vous utilisez Reacts Windows, vous devrez entrer manuellement ces informations dans la section “Paramètres réseau” de l’application Reacts.

1. Lancer Reacts et cliquer sur “Paramètre réseau” dans la page de connexion.

Note: Reacts WebApp utilisera les mêmes paramètres que ceux présents dans les paramètres de Chrome.

Reacts mobile utilisera les mêmes paramètres que ceux présents dans les paramètres de votre téléphones.

La majorité des échecs d’installation apparaîssent lorsque des mises-à-jours Windows sont en attentes ou que des librairies sont expirées. Par conséquent, veuillez vous assurer de ne pas avoir de mises à jours Windows en attentes.

Pour s’assurer que les librairies Windows sont à jours, suivre les étapes détaillées dans notre support en ligne, que vous pourrez trouver dans les documentations suivantes: 

- “Message d’erreur Microsoft .NET Framework en essayant d’installer Reacts”

- “Les messages “Reacts n’a pas pu accéder aux serveurs» ou « Impossible de charger le fichier d’assemblage Name.dll ou l’une de ses composantes » s’affichent lorsque j’essaie d’ouvrir une session Reacts pour la première fois après l’installation”
