Installation de l’utilisateur

L’installeur Reacts pour l’utilisateur final se présente sous la forme d’un exécutable appelé “Setup.exe”. Il permet la vérification et l’installation des prérequis nécessaire à Reacts, tout comme la validation de la version et des autres tâches potentiellement préalablement nécessaire à l’installation de l’application.

Note: Reacts nécessite des droits élevés pour être installé sur l’ordinateur souhaité. L’absence de droit d’administrateur sur un ordinateur donné fera échouer l’installation de Reacts.

Installation par domaine

Pour les administrateurs de domaine, il peut être préférable d’utiliser notre alternative avec Windows Installer (MSI). Dans ce cas, les administrateurs de domaine peuvent pré-configurer l’installation pour le domaine afin d’accorder les droits nécessaires d’installation au lieu d’avoir à accorder l’accès à la personne. Ceci est réalisable en ajoutant le package MSI “Reacts.msi” dans la politique de stratégie de groupe d’installation des logiciels dans le domaine.*

*Se référer à la documentation présente sur le site internet de microsoft afin d’adapter cette installation à votre propre version et à votre environnement: https://technet.microsoft.com/fr-fr/library/cc753792(v=ws.11).aspx

Note: En raison de la non-installation des prérequis par le package msi, il est de la responsabilité de l’administrateur système de s’assurer que tous les prérequis sont disponibles sur l’ordinateur souhaité préalablement au déploiement de Reacts en utilisant le fichier .msi

Prérequis

Ci-dessous, vous trouverez la liste des 3 prérequis à installer. Ceux-ci sont tous des outils standards de Microsoft et peuvent être téléchargeables directement sur le site de Microsoft.

.NET Framework 4.6.2 ou supérieur

Redistribuable Visual C++ pour Visual Studio 2013 (Veuillez installer les versions X64 bit et X86 bit)

Redistribuable Visual C++ pour Visual Studio 2015 (Veuillez installer les versions X64 bit et X86 bit)

Installation silencieuse et pilote d’encryption

Durant l’installation, Reacts installe un disque virtuel afin de permettre l’encryption locale des documents de Reacts. Durant l’installation classique de Reacts, l’utilisateur est invité à installer ce disque. Depuis la version 2.10 de Reacts cette installation et par conséquent l’encryption locale de vos données est obligatoire. Vous trouverez ci-dessous des exemples permettant d’éviter l’invite pour l’installation du pilote d’encryption:

 c:\windows\system32\msiexec.exe /i path-to-reacts.msi /quiet /qn /norestart /log path-to-install.log USECBFS=1

c:\windows\system32\msiexec.exe - Chemin d’accès par défaut de l’installeur Windows

/i path-to-reacts.msi – Installation du fichier spécifié 

/quiet - Mode silencieux (sans interaction avec l’utilisateur)

/qn – Sans information pour l’utilisateur

/norestart -  La machine ne va pas redémarrer après la fin de l’installation

/log  path-to-install.log – Chemin d’accès du fichier contenant les logs de l’installation

USECBFS=1 – Installation du pilote d’encryption

INSTALLGIVECONTROL=0 – Ne pas installer la fonctionnalité de donner et prendre le contrôle du partage d’écran

INSTALLSHAREDISPLAY=0 – Ne pas installer la fonctionnalité de partage d’écran et ses sous fonctionnalités

 i.e.:C:\WINDOWS\system32>msiexec /i c:\temp\reacts.msi /quiet /qn /norestart /log c:\temp\install.log USECBFS=1

Si le certificat Eldos n’est pas accepté préalablement, Windows affichera un message de sécurité similaire à celui-ci :

La non installation de ce driver empêche l’installation de la plateforme Reacts!

La manière d’éviter ce message est d’installer le certificat EldoS dans la banque de certificats. Vous pouvez télécharger ce certificat ici. Installer ce certificat à la main ou en utilisant une politique de groupe.

Si vous préférez installer ce certificat en tant que certificat utilisateur, et non en tant que certificat d’ordinateur, assurez-vous que ce même utilisateur sera l’installateur.

Voici les messages qui s’afficheront si vous cliquez sur “Don’t Install” :

Installation du certificat via une politique de groupe

Ouvrir l’Éditeur de Gestion des Stratégies de Groupes. 

Éditer l’Objet de Stratégie de Groupes que vous souhaitez mettre à jour ou en créer un nouveau.

Aller à “Configuration ordinateur\Paramètres Windows\Paramètres de sécurité\Stratégies de clé publique\Éditeurs approuvés” et y importer le certificat téléchargé.

S’assurer que l’Objet de Stratégie de Groupes est mis à jour/appliqué pour l’ensemble des ordinateurs/Utilisateurs qui utilise Reacts.

Une fois le certificat installé, l’installeur silencieux de Reacts devrait fonctionner silencieusement comme attendu.
